# Platform Freeze Declaration — Sprint 8.5

**Date:** 2026-06-30  
**Release:** Release 2  
**Version:** v1.0

---

## Declaration

The iDoo ERP Platform (`src/app/core/platform/`) is hereby declared **frozen at v1.0**.

All subsystems from Sprint 2 through Sprint 8.4 have been audited, stabilized, and documented. The platform is approved for use as the foundation of the Dynamic Table Engine and subsequent ERP modules.

---

## What Is Frozen

### Source Code

- All files in `src/app/core/platform/` as of Sprint 8.5.
- No refactoring of frozen files without an Architecture Review.
- Public API surface documented in `04_PUBLIC_API_FREEZE.md` must not change.

### Injection Tokens

All `InjectionToken` values in `*.tokens.ts` files are frozen. New tokens may be added. Existing tokens must not be renamed or have their default factory changed.

### CSS Custom Properties

All `--platform-*` CSS variables are frozen. New variables may be added. Existing names must not be removed or renamed.

### HTML Attributes

All `data-*` attributes applied by platform engines are frozen. New attributes may be added.

---

## What Is Not Frozen

The following are expected to grow in future sprints:

| Item | Expected Change |
|---|---|
| `VisualExperienceRegistryService` | New profiles registered by tenants at runtime |
| `TranslationRegistryService` | New namespaces/locales loaded at runtime |
| `MetadataEngineService` | New entity types registered by ERP modules |
| `ComponentRegistryService` | New field components added for Dynamic Table |
| `ThemeRegistryService` | New tenant themes registered at runtime |
| `RendererRegistryService` | New renderers for Dynamic Table cells |
